# CLAUDE.md

## Git rules

- **Never `git commit` unless explicitly asked in that same message.**
- **Never `git push` unless explicitly asked in that same message.**
- Applies to all variants (`commit -am`, `--amend`, `push --force`, aliases, scripts).
- Approval doesn't carry over — ask again each time.
- When unsure, assume "no": leave changes uncommitted and say it's ready.
